# Project Yojanā – Onboarding & Training Manual

## 1. Introduction
This manual provides onboarding instructions and training guidelines for faculty, exam coordinators, and administrators using Project Yojanā. It ensures smooth adoption of blueprint-driven paper generation, compliance validation, and secure download workflows.

## 2. Audience
- **Faculty:** Create and validate exam papers using blueprints.
- **Exam Coordinators:** Manage paper readiness, oversee compliance validation.
- **Administrators:** Ensure accreditation compliance and oversee secure distribution.

## 3. Prerequisites
- Valid institutional login credentials.
- Role-based access assigned by administrator.
- Basic familiarity with exam blueprints, Bloom’s taxonomy, and CO mappings.

## 4. Onboarding Steps
1. **Login:** Access Yojanā via institutional portal with secure credentials.
2. **Role Verification:** Confirm dashboard access matches assigned role.
3. **Profile Setup:** Faculty configure blueprint preferences; coordinators set compliance rules.
4. **Initial Training:** Attend orientation session (1 hour) covering paper generation and compliance workflows.

## 5. Training Modules
- **Module 1: Paper Generation**
  - Faculty upload blueprint metadata.
  - Generate exam papers with automated engine.
  - Review paper variants for accuracy.

- **Module 2: Compliance Validation**
  - Run compliance checker for Bloom’s taxonomy and CO mappings.
  - Interpret flagged issues.
  - Adjust blueprints to meet accreditation standards.

- **Module 3: Secure Download**
  - Coordinators approve papers for release.
  - Papers watermarked and metadata applied.
  - Administrators oversee secure distribution.

- **Module 4: Integration Awareness**
  - Papers archived in Sangrah.
  - Evaluated in Viveka.
  - Analytics tracked in Anvesha.

## 6. FAQs
- **Q:** What if compliance checker flags errors?  
  **A:** Review blueprint metadata and adjust mappings before re-running validation.  
- **Q:** Can papers be generated in multiple variants?  
  **A:** Yes, the engine supports variant generation.  
- **Q:** How are papers secured?  
  **A:** Papers are watermarked and logged with metadata before download.

## 7. Troubleshooting
- **Issue:** Login failure → Contact IT admin for credential reset.  
- **Issue:** Paper generation fails → Verify blueprint metadata completeness.  
- **Issue:** Compliance checker not running → Ensure CO mappings are uploaded correctly.  

## 8. Training Schedule
- Orientation: Week 1 (all roles).
- Faculty paper generation workshop: Week 2.
- Coordinator compliance deep-dive: Week 3.
- Administrator accreditation session: Week 4.

## 10. Continuous Learning
- Quarterly refresher workshops.
- Updated training materials with each release.
- Feedback loop for improving onboarding experience.
