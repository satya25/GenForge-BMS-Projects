# Project Viveka – Maintenance & Support Plan

## 1. Introduction
This document defines the maintenance and support strategy for Project Viveka. It ensures long-term reliability of automated grading, evaluation workflows, and archival of evaluated papers, while supporting faculty, exam coordinators, and administrators.

## 2. Objectives
- Maintain grading engine accuracy and responsiveness.
- Provide timely support for faculty and coordinators.
- Ensure compliance with institutional evaluation standards.
- Continuously improve consistency checks and reporting features.

## 3. Maintenance Activities
- **Routine Maintenance:**
  - Weekly server health checks.
  - Monthly database optimization.
  - Quarterly security audits.
- **Corrective Maintenance:**
  - Bug fixes within SLA timelines.
  - Immediate resolution of grading engine errors.
- **Adaptive Maintenance:**
  - Updates to support new evaluation policies and question formats.
  - Enhancements to consistency checker and dashboards.
- **Preventive Maintenance:**
  - Proactive monitoring of grading throughput.
  - Regular patching of backend services.

## 4. Support Structure
- **Tier 1:** Helpdesk support for login, dashboard navigation, and evaluation access.
- **Tier 2:** Technical support for grading engine and consistency checker.
- **Tier 3:** System admin support for integration issues and archival workflows.

## 5. SLA Commitments
- **Critical Issues:** Resolution within 24 hours.
- **Major Issues:** Resolution within 72 hours.
- **Minor Issues:** Resolution within 7 days.
- **Uptime Guarantee:** ≥ 99.5% availability.

## 6. Monitoring & Alerts
- Prometheus + Grafana dashboards for performance monitoring.
- ELK stack for centralized logging.
- Automated alerts for downtime, grading failures, or consistency errors.

## 7. Escalation Matrix
- Faculty → Helpdesk (Tier 1).
- Coordinators → Technical Support (Tier 2).
- Administrators → System Admin (Tier 3).
- Escalation to project owner if SLA breached.

## 8. Documentation & Knowledge Base
- User guides updated quarterly.
- FAQs maintained in institutional portal.
- Troubleshooting playbooks for common grading issues.

## 9. Continuous Improvement
- Quarterly feedback sessions with faculty and coordinators.
- Annual review of grading engine accuracy.
- Iterative enhancements to consistency checker and dashboards.

## 10. Maintenance Schedule
- Weekly: Server health checks.
- Monthly: Database optimization.
- Quarterly: Security audits & feedback review.
- Annual: Evaluation policy compliance update.
