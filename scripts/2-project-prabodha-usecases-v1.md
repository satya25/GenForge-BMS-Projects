## 1. Actors
- **Students:** Access guided practice modules, receive feedback, and track progress.
- **Faculty:** Monitor student learning outcomes and engagement.
- **HODs:** Oversee department-level adoption and effectiveness.
- **Academic Council:** Evaluate learning impact for curriculum improvements.

## 2. Use Cases
### UC-1: Student Practice Session
- **Actor:** Student
- **Precondition:** Student logged in with valid credentials.
- **Steps:**
  1. Navigate to practice module.
  2. Select subject/topic.
  3. Attempt questions (MCQs, short answers, problem-solving).
  4. Receive instant feedback and hints.
- **Postcondition:** Student completes adaptive practice session.

### UC-2: Faculty Progress Monitoring
- **Actor:** Faculty
- **Precondition:** Students have attempted practice sessions.
- **Steps:**
  1. Faculty logs in.
  2. Selects course/subject.
  3. Views student progress dashboard.
  4. Identifies weak areas and learning gaps.
- **Postcondition:** Faculty gains insights into student performance.

### UC-3: Adaptive Difficulty Adjustment
- **Actor:** System
- **Precondition:** Student attempts multiple questions.
- **Steps:**
  1. System evaluates student performance.
  2. Adjusts difficulty level dynamically.
  3. Presents next set of questions accordingly.
- **Postcondition:** Student receives personalized learning pathway.

## 3. User Stories
- As a **student**, I want adaptive practice sessions so that I can improve my learning outcomes.  
- As a **faculty member**, I want to monitor student progress so that I can identify weak areas.  
- As a **HOD**, I want to oversee adoption metrics so that I can ensure department-level effectiveness.  

## 4. Workflows
### Workflow 1: Student Practice Session
1. Student logs in.
2. Selects subject/topic.
3. System presents questions.
4. Student attempts questions.
5. System provides feedback and hints.
6. Student tracks progress.

### Workflow 2: Faculty Monitoring
1. Faculty logs in.
2. Selects course/subject.
3. Dashboard displays student progress and analytics.
4. Faculty identifies weak areas.
5. Faculty plans interventions.

### Workflow 3: Adaptive Difficulty
1. Student attempts initial set of questions.
2. System evaluates performance.
3. Difficulty level adjusted dynamically.
4. Next set of questions presented.
5. Student continues practice with adaptive pathway.
