# Project Sangrah – Onboarding & Training Manual

## 1. Introduction
This manual provides onboarding instructions and training guidelines for faculty, HODs, and administrators using Project Sangrah. It ensures smooth adoption of archival, retrieval, and version control features that support accreditation and compliance.

## 2. Audience
- **Faculty:** Archive papers and blueprints, retrieve usage history.
- **HODs:** Review archival compliance, monitor subject-level document integrity.
- **Administrators:** Oversee archival processes, ensure accreditation readiness.

## 3. Prerequisites
- Valid institutional login credentials.
- Role-based access assigned by administrator.
- Basic familiarity with exam papers, blueprints, and accreditation requirements.

## 4. Onboarding Steps
1. **Login:** Access Sangrah via institutional portal with secure credentials.
2. **Role Verification:** Confirm dashboard access matches assigned role.
3. **Profile Setup:** Update archival preferences and notification settings.
4. **Initial Training:** Attend orientation session (1 hour) covering archival workflows and retrieval engine.

## 5. Training Modules
- **Module 1: Archival Workflows**
  - Faculty upload papers and blueprints.
  - HODs verify archival compliance.
  - Administrators oversee archival logs.

- **Module 2: Retrieval Engine**
  - Search for archived papers by subject, date, or usage history.
  - Generate reports on question usage.
  - Export retrieval logs for accreditation.

- **Module 3: Version Control**
  - Track document versions.
  - Rollback to previous versions if required.
  - Validate integrity with audit logs.

- **Module 4: Integration Awareness**
  - Understand paper flow from Yojanā.
  - Review evaluated papers from Viveka.
  - Access analytics in Anvesha.

## 6. FAQs
- **Q:** How do I confirm a paper is archived?  
  **A:** Check archival logs and metadata in the dashboard.  
- **Q:** Can I retrieve usage history for accreditation?  
  **A:** Yes, retrieval engine provides detailed usage reports.  
- **Q:** What if version control shows errors?  
  **A:** Contact system admin to validate rollback integrity.  

## 7. Troubleshooting
- **Issue:** Login failure → Contact IT admin for credential reset.  
- **Issue:** Paper not visible → Verify archival status.  
- **Issue:** Retrieval engine slow → Run search with refined filters.  

## 8. Training Schedule
- Orientation: Week 1 (all roles).
- Faculty archival workshop: Week 2.
- HOD compliance deep-dive: Week 3.
- Administrator accreditation session: Week 4.

## 9. Support Channels
- Helpdesk email: support@institute.edu
- Dedicated Slack/Teams channel for queries.
- Monthly Q&A sessions with system admins.

## 10. Continuous Learning
- Quarterly refresher workshops.
- Updated training materials with each release.
- Feedback loop for improving onboarding experience.
