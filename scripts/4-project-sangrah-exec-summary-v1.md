# Project Sangrah – Executive Summary

## 1. Vision & Purpose
Sangrah is designed as the institute’s data consolidation and archival system. Its purpose is to unify academic records, exam metadata, and question bank usage into a single, structured repository. Sangrah ensures that faculty and administrators can access reliable, versioned data for audits, compliance, and long-term institutional memory.

## 2. Objectives
- Consolidate exam papers, blueprints, and question usage history into a unified database.
- Provide version-controlled archival of academic documents.
- Enable faculty and administrators to trace question usage across semesters.
- Support compliance with accreditation and audit requirements.
- Integrate with Samaya (scheduling) and Anvesha (analytics).

## 3. Scope
**In-Scope:**
- Centralized repository for exam papers and blueprints.
- Question usage tracking and archival.
- Faculty and HOD dashboards for data retrieval.
- Integration with Yojanā for generated papers.

**Out-of-Scope:**
- Real-time analytics (handled by Anvesha).
- Paper generation (handled by Yojanā).
- Student-facing portals.

## 4. Stakeholders
- **Faculty:** access historical papers and question usage.
- **HODs:** oversee department-level archival and compliance.
- **Exam Coordinators:** trace paper variants and downloads.
- **Administrators:** manage institute-wide archival policies.
- **Academic Council:** review consolidated data for accreditation.

## 5. Key Features
- Centralized archival of papers, blueprints, and metadata.
- Question usage history with anti-repetition tracking.
- Version-controlled storage of academic documents.
- Secure access with audit logs.
- Integration with Yojanā and Anvesha for seamless data flow.

## 6. Success Criteria
- 100% archival of generated papers and blueprints.
- Accurate question usage tracking across semesters.
- Zero discrepancies in audit reports.
- ≥80% faculty adoption for archival retrieval.
- Seamless integration with Yojanā and Anvesha.

## 7. Roadmap Snapshot
- **MVP:** Centralized repository for papers and blueprints.
- **Phase I:** Question usage tracking and dashboards.
- **Phase II:** Integration with Yojanā and Anvesha.
- **Phase III:** Institute-wide archival with predictive insights.
