# Project Samaya – Onboarding & Training Manual

## 1. Introduction
This manual provides onboarding instructions and training guidelines for faculty, coordinators, administrators, and students using Project Samaya. It ensures smooth adoption of academic calendar creation, exam scheduling, and deadline tracking features.

## 2. Audience
- **Faculty:** Track deadlines, receive reminders, manage exam schedules.
- **Coordinators:** Create and publish academic calendars, resolve scheduling conflicts.
- **Administrators:** Oversee scheduling compliance and institutional timelines.
- **Students:** View published calendars and exam schedules.

## 3. Prerequisites
- Valid institutional login credentials.
- Role-based access assigned by administrator.
- Basic familiarity with academic events and exam scheduling processes.

## 4. Onboarding Steps
1. **Login:** Access Samaya via institutional portal with secure credentials.
2. **Role Verification:** Confirm dashboard access matches assigned role.
3. **Profile Setup:** Faculty set notification preferences; coordinators configure scheduling rules.
4. **Initial Training:** Attend orientation session (1 hour) covering calendar creation and scheduling workflows.

## 5. Training Modules
- **Module 1: Calendar Creation**
  - Coordinators create academic events.
  - Faculty review deadlines.
  - Administrators approve institutional calendars.

- **Module 2: Exam Scheduling**
  - Coordinators schedule exams.
  - Conflict detection engine highlights overlaps.
  - Faculty verify exam readiness.

- **Module 3: Deadline Tracking**
  - Faculty receive reminders for paper readiness.
  - Coordinators monitor submission timelines.
  - Administrators track compliance.

- **Module 4: Integration Awareness**
  - Understand paper readiness flow from Yojanā.
  - Review evaluation timelines in Viveka.
  - Access analytics in Anvesha.

## 6. FAQs
- **Q:** What if scheduling conflicts occur?  
  **A:** Conflict detection engine highlights overlaps; coordinators must resolve before publishing.  
- **Q:** Can students access calendars?  
  **A:** Yes, published calendars are visible to students via the portal.  
- **Q:** How are reminders delivered?  
  **A:** Via email and in-app notifications.

## 7. Troubleshooting
- **Issue:** Login failure → Contact IT admin for credential reset.  
- **Issue:** Calendar not visible → Verify publishing status.  
- **Issue:** Reminders not received → Check notification preferences.  

## 8. Training Schedule
- Orientation: Week 1 (all roles).
- Faculty deadline tracking workshop: Week 2.
- Coordinator scheduling deep-dive: Week 3.
- Administrator compliance session: Week 4.

## 9. Support Channels
- Helpdesk email: support@institute.edu
- Dedicated Slack/Teams channel for queries.
- Monthly Q&A sessions with system admins.

## 10. Continuous Learning
- Quarterly refresher workshops.
- Updated training materials with each release.
- Feedback loop for improving onboarding experience.
