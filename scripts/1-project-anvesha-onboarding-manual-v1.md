# Project Anvesha – Onboarding & Training Manual

## 1. Introduction
This manual provides onboarding instructions and training guidelines for faculty, HODs, and administrators using Project Anvesha. It ensures smooth adoption of analytics, compliance validation, and reporting features.

## 2. Audience
- **Faculty:** Validate blueprint compliance, monitor dashboards.
- **HODs:** Review compliance reports, track subject-level performance.
- **Administrators:** Access accreditation-ready reports, oversee system usage.

## 3. Prerequisites
- Valid institutional login credentials.
- Role-based access assigned by administrator.
- Basic familiarity with exam blueprints and CO mappings.

## 4. Onboarding Steps
1. **Login:** Access Anvesha via institutional portal with secure credentials.
2. **Role Verification:** Confirm dashboard access matches assigned role.
3. **Profile Setup:** Update contact details and notification preferences.
4. **Initial Training:** Attend orientation session (1 hour) covering dashboards and compliance checker.

## 5. Training Modules
- **Module 1: Dashboard Navigation**
  - Faculty view: compliance metrics, flagged papers.
  - HOD view: subject-level compliance summaries.
  - Admin view: accreditation-ready reports.

- **Module 2: Compliance Validation**
  - Upload blueprint metadata.
  - Run compliance checker.
  - Interpret Bloom’s taxonomy and CO mapping results.

- **Module 3: Reporting**
  - Generate compliance reports.
  - Export reports for accreditation.
  - Archive reports in Sangrah.

- **Module 4: Integration Awareness**
  - Understand data flow from Yojanā (papers).
  - Review evaluation results from Viveka.
  - Access analytics dashboards in Anvesha.

## 6. FAQs
- **Q:** What if compliance checker flags errors?  
  **A:** Review blueprint metadata and re-run validation.  
- **Q:** Can reports be exported?  
  **A:** Yes, reports can be exported in PDF/Excel formats.  
- **Q:** How often should dashboards be checked?  
  **A:** Faculty should check after each exam cycle; HODs monthly; administrators quarterly.

## 7. Troubleshooting
- **Issue:** Login failure → Contact IT admin for credential reset.  
- **Issue:** Missing dashboard features → Verify role assignment.  
- **Issue:** Compliance checker not running → Ensure blueprint metadata is uploaded correctly.  

## 8. Training Schedule
- Orientation: Week 1 (all roles).
- Faculty deep-dive: Week 2.
- HOD workshop: Week 3.
- Administrator accreditation session: Week 4.

## 9. Support Channels
- Helpdesk email: support@institute.edu
- Dedicated Slack/Teams channel for queries.
- Monthly Q&A sessions with system admins.

## 10. Continuous Learning
- Quarterly refresher workshops.
- Updated training materials with each release.
- Feedback loop for improving onboarding experience.
