# Project Prabodha – Maintenance & Support Plan

## 1. Introduction
This document defines the maintenance and support strategy for Project Prabodha. It ensures long-term reliability of adaptive practice sessions, feedback generation, and monitoring dashboards, while supporting students, faculty, and administrators.

## 2. Objectives
- Maintain adaptive engine accuracy and responsiveness.
- Provide timely support for students and faculty.
- Ensure compliance with institutional learning outcomes.
- Continuously improve dashboards and feedback mechanisms.

## 3. Maintenance Activities
- **Routine Maintenance:**
  - Weekly server health checks.
  - Monthly database optimization.
  - Quarterly security audits.
- **Corrective Maintenance:**
  - Bug fixes within SLA timelines.
  - Immediate resolution of adaptive engine errors.
- **Adaptive Maintenance:**
  - Updates to support new question banks and CO mappings.
  - Enhancements to feedback generation templates.
- **Preventive Maintenance:**
  - Proactive monitoring of student progress logs.
  - Regular patching of backend services.

## 4. Support Structure
- **Tier 1:** Helpdesk support for login, practice session access, and feedback review.
- **Tier 2:** Technical support for adaptive engine and faculty dashboards.
- **Tier 3:** System admin support for integration issues and institutional compliance.

## 5. SLA Commitments
- **Critical Issues:** Resolution within 24 hours.
- **Major Issues:** Resolution within 72 hours.
- **Minor Issues:** Resolution within 7 days.
- **Uptime Guarantee:** ≥ 99.5% availability.

## 6. Monitoring & Alerts
- Prometheus + Grafana dashboards for performance monitoring.
- ELK stack for centralized logging.
- Automated alerts for downtime, adaptive engine failures, or feedback generation errors.

## 7. Escalation Matrix
- Students → Helpdesk (Tier 1).
- Faculty → Technical Support (Tier 2).
- Administrators → System Admin (Tier 3).
- Escalation to project owner if SLA breached.

## 8. Documentation & Knowledge Base
- User guides updated quarterly.
- FAQs maintained in institutional portal.
- Troubleshooting playbooks for common issues.

## 9. Continuous Improvement
- Quarterly feedback sessions with students and faculty.
- Annual review of adaptive engine accuracy.
- Iterative enhancements to feedback generation and dashboards.

## 10. Maintenance Schedule
- Weekly: Server health checks.
- Monthly: Database optimization.
- Quarterly: Security audits & feedback review.
- Annual: Learning outcome compliance update.
