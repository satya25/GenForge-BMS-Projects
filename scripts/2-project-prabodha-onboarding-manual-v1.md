# Project Prabodha – Onboarding & Training Manual

## 1. Introduction
This manual provides onboarding instructions and training guidelines for students, faculty, and administrators using Project Prabodha. It ensures smooth adoption of adaptive practice sessions, feedback generation, and monitoring dashboards.

## 2. Audience
- **Students:** Engage in adaptive practice sessions, receive feedback.
- **Faculty:** Monitor student progress, review feedback accuracy.
- **Administrators:** Oversee adoption and ensure compliance with institutional learning outcomes.

## 3. Prerequisites
- Valid institutional login credentials.
- Role-based access assigned by administrator.
- Basic familiarity with subject question banks and CO mappings.

## 4. Onboarding Steps
1. **Login:** Access Prabodha via institutional portal with secure credentials.
2. **Role Verification:** Confirm dashboard access matches assigned role.
3. **Profile Setup:** Students update learning preferences; faculty set monitoring parameters.
4. **Initial Training:** Attend orientation session (1 hour) covering adaptive engine and dashboards.

## 5. Training Modules
- **Module 1: Student Practice Sessions**
  - Start adaptive practice sessions.
  - Review feedback after each session.
  - Track personal progress metrics.

- **Module 2: Adaptive Engine**
  - Understand difficulty scaling.
  - Learn how feedback is generated.
  - Recognize how CO mappings influence practice.

- **Module 3: Faculty Dashboard**
  - Monitor student progress.
  - Review flagged feedback cases.
  - Export progress reports for analysis.

- **Module 4: Integration Awareness**
  - Understand question bank flow from Yojanā.
  - Review analytics in Anvesha.
  - Align practice outcomes with institutional goals.

## 6. FAQs
- **Q:** How does the adaptive engine adjust difficulty?  
  **A:** Based on student performance and CO mappings.  
- **Q:** Can faculty override feedback?  
  **A:** Yes, faculty can review and adjust feedback templates.  
- **Q:** How often should students practice?  
  **A:** At least 3 sessions per week for optimal learning.

## 7. Troubleshooting
- **Issue:** Login failure → Contact IT admin for credential reset.  
- **Issue:** Feedback not generated → Ensure adaptive engine is active.  
- **Issue:** Dashboard not updating → Refresh session or contact support.  

## 8. Training Schedule
- Orientation: Week 1 (all roles).
- Student practice workshop: Week 2.
- Faculty dashboard deep-dive: Week 3.
- Administrator compliance session: Week 4.

## 9. Support Channels
- Helpdesk email: support@institute.edu
- Dedicated Slack/Teams channel for queries.
- Monthly Q&A sessions with system admins.

## 10. Continuous Learning
- Quarterly refresher workshops.
- Updated training materials with each release.
- Feedback loop for improving onboarding experience.
